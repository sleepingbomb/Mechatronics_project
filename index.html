Closed-Loop Motor Control System

Development and validation of a closed-loop motor position control system on an Arduino-based platform with P, PD, and PID controllers for DC motor tracking.

Project Overview

The goal of this project is to develop and validate a closed-loop motor position control system on an Arduino-based platform that can accurately track a desired reference signal. By designing Simulink models and implementing P, PD, and PID controllers for a DC motor with quadrature encoder feedback, the project aims to systematically tune and compare the three control schemes.

The evaluation focuses on three key performance metrics: stability, steady-state error, and dynamic response. The objective is to demonstrate how adding derivative and integral actions improves tracking performance and reduces error, resulting in a more precise and robust motion control system.

Simulink Model Architecture

The overall Simulink model implements a closed-loop position control system for a DC motor using an Arduino board and a quadrature encoder. The system is composed of six main functional blocks:

Direction Control Block

Sets the rotation direction using digital outputs on pins 4 and 5.

  • Logic constants (uint8 format)
  • [1,0] = CW, [0,1] = CCW

Reference Generator Block

Defines the target position trajectory.

  • Sine wave generator

PWM Output Block

Converts control effort to PWM signal.

  • Arduino PWM Output (Pin 9)
  • Duty cycle: 0-255 range
  • Motor driver interface

Controller Operation

The same closed-loop Simulink model evaluates three different control strategies. The plant (DC motor + driver + encoder) and feedback structure remain identical; only the controller gains change between experiments.

P Proportional Control

Only the proportional gain (Kp) is enabled, with integral and derivative gains set to zero. The controller output is directly proportional to the instantaneous position error:

u(t) = Kp Â· e(t)

Higher Kp reduces tracking error and speeds up response, but excessive values cause overshoot and oscillation. With P control alone, a non-zero steady-state error is generally expected.

PD Proportional-Derivative Control

Both proportional (Kp) and derivative (Kd) gains are used. The controller reacts to both the magnitude and rate of change of the error:

u(t) = Kp Â· e(t) + Kd Â· de(t)/dt

The derivative action predicts future error behavior and counteracts rapid changes, reducing overshoot and oscillations. PD control provides faster, more stable transient response but may still exhibit steady-state error.

The integral term accumulates error over time and forces it toward zero, eliminating the steady-state offset present in P and PD control. PID provides the best overall tracking performance.

Conclusion

P
Proportional Control

Simple to implement and achieves basic tracking, but typically leaves noticeable steady-state error and may cause oscillations when gain is increased too much.

PD
Proportional-Derivative Control

Significantly improves transient response by reducing overshoot and oscillations through derivative damping, but still cannot completely remove steady-state error.

PID
Proportional-Integral-Derivative Control

Provides the best overall performanceâ€”integral action drives steady-state error close to zero while maintaining acceptable rise time and stability when properly tuned.

Key Takeaway

This project successfully demonstrates how the addition of derivative and integral terms to a proportional controller can enhance tracking accuracy and robustness in a practical motor control system. The systematic comparison of P, PD, and PID controllers provides clear evidence of the trade-offs involved in tuning real-world control systems.
